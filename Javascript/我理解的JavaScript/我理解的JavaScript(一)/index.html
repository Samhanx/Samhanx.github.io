我理解的JavaScript(一)

发表于 2017-04-26 | 分类于 Javascript ， 我理解的JavaScript

如果不算中学时只是玩过一小会的VB，不算超文本标记语言HTML和层叠样式表CSS，JavaScript应该是我正式学习，运用，并且小有所得的第一门编程语言（这里没有任何看低HTML和CSS的意思，实际上对作为前端的我而言，这两者也非常重要）。一直以来我都想写一些我在学习JS时的理解，笔记和总结，但是从开始工作以后，JS的学习就变得不那么系统了，各种零碎的知识点穿插在日常工作和学习的积累当中，如果一直这样放任下去，不加以整理归纳，绝非好事。因此我决定陆续开始写这样一个系列的文章，名字叫做“我理解的JavaScript”，把我通过阅读各类JS的技术书籍，网络上的相关文章和教程，以及自己在实际工作中积累的经验，在经过学习理解之后，以文字的形式把结果记录下来，也算是达成了一直以来的想法。文章会是我自己的思考理解，因而也许会有偏差，但作为一个长期的个人写作项目，一旦知识有所更新或者纠正，我也会对自己写过的内容进行相应的调整。

JS引擎和ECMAScript

简单说说JS引擎

JS被认为是一种解释型的语言，简单来说，它不像C，Java，Python等等语言那样需要我们在代码执行之前进行编译，而直接由解释器解析并执行。我们通常在使用JS的时候，也是以脚本的形式插入到HTML中，浏览器打开就能看到代码的运行结果，或者直接在浏览器的调试台里输入JS代码然后直接运行。这个浏览器里面其实就实现了一个JS的解释引擎，使得它可以完成这样的工作，比如其中非常有名的Chrome的V8引擎。

编译器是将源代码编译为另外一种代码（比如机器码，或者字节码），而解释器是直接解析并将代码运行结果输出。

然而即便是作为解释执行的语言，JS引擎也会对JS代码进行一个编译的过程，而且这个过程与代码的执行紧密地结合在一起，以至于我们几乎无法感知到编译的时间，因为通常一段可执行的JS代码片段在编译之后程序立刻就会执行它（将它推入执行栈空间）。

个人认为，要深入理解JS，则必须要理解JS的源码从编译到执行所经历的这样一个过程，所以我们需要先树立起JS引擎这样一个概念。而这其中必然还要谈到另一个我们常常见到的名词——ECMAScript(下简称ES)。

ECMA是欧洲计算机制造商协会(European Computer Manufacturers Association)的首字母简称，ES是它制定的一套语言标准。

我们所学习的JavaScript，几乎可以说是以浏览器作为宿主来谈论的（暂不谈Node.js）。其中包括了JS语言本身的标准，即ES、文档对象模型（Document Object Model）DOM、浏览器对象模型（Browser Object Model）BOM，这三个部分的内容。一个标准的JS解释引擎，就是参照语言标准ES，用代码对它进行编写实现，这个标准定义了语言的词法，语法，规范等这些规则性质的内容。比方说变量的声明和赋值如何解析var a = 1;，定义数据类型、操作和运算，以及控制结构if else等等内容。具体包括了语法，类型，语句，关键字，保留字，操作符，对象，原生API这几个方面。

放在浏览器的JS引擎来说，除了要加入DOM和BOM相关的内容之外，各个厂商也会结合自家产品在具体的API上或新增或修改，并且对ES这个统一标准的实现程度也各有不同，所以也就导致了JS在不同浏览器下运行会有兼容性的问题。

幸运的是，随着制定标准的委员会官方和各个web技术社区的推动，以及浏览器市场的变化，现在各家浏览器厂商对于ES标准的实现正在趋于一致，JS的兼容性问题如今几乎已经成为历史，更多的前沿技术甚至宣称不再支持IE9及其以下的老旧浏览器，而Babel的出现也能够让我们在项目中运用最新的ES标准来编写代码，这是非常令人可喜的事情。

ECMAScript简要历史

详细的历史可以通过Google查阅相关资料，这里仅就我认为的一些关键点进行总结。

  • JavaScript之父Brendan Eich于1995年创造了这门语言。

  • 1997年ECMA委员会以JS1.1版本作为蓝本制定了第一版ECMAScript，并于当年7月发布，名为ECMA-262(第262号文件)。TC-39是负责制定标准的技术委员会。

  • 1999年发布的ECMA-262第三版也是极其重要的一个版本，实际上在我们看来理所当然的如正则表达式，try-catch语句，Error类型等内容都是这一版加入进去的。而IE6-IE8浏览器的JS引擎也是实现的这一版本，所以要兼容这些浏览器，不得不放弃很多有用的原生API。

  • 2009年，事隔十年之后，ECMA-262标准第五版发布。IE9及以后的浏览器，以及大部分现代浏览器都以这一版标准实现JS引擎。第五版标准大量拓展了String，Array，Object，Date等对象上的方法。

  • 在第五版之前曾有一个第四版的草案，因为太过激进，各方博弈之下最终没有发布，但实际上其中很多内容在随后的ES5和后来的ES6中进行了实现。

  • 2015年6月，几经讨论的ES6标准最终发布，并且ECMA委员会决定之后以年份为标记，每年6-7月发布当年的版本。
