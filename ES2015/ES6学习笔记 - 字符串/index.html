ES6学习笔记 - 字符串

发表于 2017-07-08 | 分类于 ES2015

ES6关于字符串的新增内容最主要也最常用的就是模板字符串了。

模板字符串

书写字符串要解决的最核心的两个问题是变量的插入以及换行。

字符串拼接的常用手段

在书写过长的字符串并结合变量的插入时，通常会采用拼接的方式，比如典型的拼接HTML场景

实际上这也是非常高效的一种字符串拼接手段，另一种方式是分段将要处理的字符串存到数组之中然后再进行Join，通常用在一些需要循环生成的字符串里
此外，还有的程序员认为数组Join的方式比直接用+运算符进行拼接处理更快捷，也许以前在某些老旧的浏览器可能会有这样的情况也说不定。
但在如今的chrome下，有人实测发现是直接拼接速度更快。其实这看上去也是完全合理的，我认为操作符层面的计算理应比函数层面的计算更加快捷。

字符串拼接解决了变量插入的问题，还有一个问题就是换行书写。在拼接中，常常会以加入换行符\n来实现换行显示。后来应该是ES3的版本，JS支持了在字符串行尾输入\实现换行。

不同于多个字符串拼接的结果，此时的str从头到尾都是一个整个字符串，如果我们进行压缩之后就会发现，\带来的换行实际上会出现一大片空格符。
换言之，这种方式其实只是在编辑器层面看上去的换行，并不适合进行HTML的拼接又或者在各种显示条件下的文本换行。
虽然HTML会将多余的空格显示为一个，但其实在代码上，由于字符串并不会进行压缩替换，所以额外的空格也会保留下来，带来空间浪费。
而在真正需要文本换行的时候，我们又常常发现只是显示了一大段空格，压根没有换行。
综合来说，不太推荐使用这种方式书写字符串。

模板字符串的出现

ES6允许我们利用反引号 ` (backtick)来书写字符串，并且可以在里面进行变量的插入和自由地换行。

${}符号内会被识别为JS代码，以上可以看到各种常见类型的变量在模板字符串中的输出结果。
我们还可以在模板字符串中进行变量的运算，甚至执行函数。

函数会取返回值转为字符串进行输出，此外${}内只接受表达式，不接受多行代码语句，注意上例执行多个函数的写法。

如果要再模板字符串执行if...else语句，则可以用三目运算符。

此外，由于可以在模板内进行函数执行，也就给我们带来了更大的操作空间。

比如模板字符串内不能直接进行流程控制和循环语句的书写，那完全可以放在函数中执行，最后返回出需要的结果即可。

string.includes(searchString[, position])

比indexOf更简单好用，但是Chrome和FF都要求比较高的版本，Opera，IE都不支持，移动端也几乎不支持

let str = ‘Hello World’;
str.includes(‘Hello’) // true
str.includes(‘hello’) // false 区分大小写
str.includes(‘H’, 1) // false 第二个参数指定检测的起点，默认为0
str.includes(‘H’, -9) // true 负数还是从0开始
