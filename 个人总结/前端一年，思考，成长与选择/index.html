前端一年，思考，成长与选择

发表于 2017-06-20 | 分类于 个人总结

2016年初，我开始正式学习前端，6月20日只身前往广州开始了前端工作，到此时刚好满一年。

经历

回顾这一年，其实收获还是挺大的，但是成长的速度却也无法令自己满意。终归还是自己不够勤奋努力，业余花在技术提升上的时间没有特别突出，很多想法都没有落地，工作和学习效率也还有很大的提升空间。

老实说，在开始从事前端工作之前的自学阶段，因为一边工作一边自学，加上其他一些杂七杂八的事情影响，效率并不算高。到最初入职的时候，技术水平可以说连很多实习生都不如，工作前几个月压力特别大，因为我所在的团队算是前后分离的开发体系，所以除了写页面之外，更多的还要写JS处理一些前端部分的业务逻辑，数据过滤以及做前后对接，现在看来驾轻就熟的一些技能点在当时我很多都没有接触过。但是压力也是动力，每天上班中午别人在休息的时候我都在看代码，思考功能和效果要如何实现；一遍遍完成功能，然后自发修改优化代码，下班时间也几乎都在晚上8点以后，回到出租屋，还继续学习工作上需要用到的技术。

在经历了最初这几个月之后，我很快就上手了公司的前端开发工作，因为上司安排的工作任务几乎是随着技术的提升一直在增加难度，所以这段时间也没有留给自己什么舒适区，刚上手了公司的常规前端开发，就开始用node.js和gulp做一些工程相关的东西，接下来又开始接触前端测试这一块，用selenium-webdriver开始写UI自动化测试脚本，可惜团队没有做前端的单元测试，这几个月应该是自己进步最快的一段时间。之后就算是进入了一个比较平稳的阶段，整个人多少懈怠了下来，上司有安排学习PHP，但其实我连语法都没有看完，后来接到完成一个比较简单的接口开发任务，临时抱佛脚看框架文档，视频学习，和同事的代码竟也顺利完成了。

从九月底开始转调到另一个项目组支援，接触了以往前后混合、嵌套PHP的开发形式，使用了sea.js，切图技能也越发熟练，并且积累了大型平台类项目的一些业务经验，但是PHP的学习却搁置了下来，没有再继续。直到年底调回团队开始独立负责起一个项目前端。

也就是去年年底两个月，自己在技术上似乎又迎来了一次比较明显的进步。其实这段时间我并没有特别花力气在技术学习上，也许是看的东西多了，加上代码量的积累，终于达到了质变，自己对于JS的理解就像捅破了一层窗户纸，之前迷迷糊糊看不太懂的作用域，闭包，原型等等这些烂大街的概念突然之间就明白了，面向对象勉强算是有了一点小心得，封装抽象代码的能力也略有提升，阅读起一些源码的障碍也少了很多。这样的成长虽然是可喜的事情，但其实来得太迟了。

早在去年八月的时候，一个偶然的机会在QQ群里接触了Free Code Camp，当时是帮别人解答一个JSON数据处理的小问题，然后觉得这种写代码答题闯关的形式比起之前慕课网接触的学习形式更加新颖有趣，而且习题安排也更利于锻炼编程思维。当然FCC也有瑕疵，比如中文翻译不太准确，知识内容还不够完善，有的项目难度跳跃太大等等，总体上瑕不掩瑜，算是一个非常适合新人实践练手的编程学习网站。只是惫懒如我断断续续做了一些题目，然后看到群里问一些感觉有意思的题才会去做一下，其实倘若全部做完想来成长也会不少，特别是其中关于算法的部分。再后来赶上FCC中国的快速发展还有各类知识付费产品的推出，加入了不少QQ群，微信群，近距离围观业界大牛，也认识了很多朋友。群里有已经从事开发工作多年的前辈大佬，也有几乎和我同期入行的新人，还有很多即将毕业的学生，以及准备伺机转行做程序员的人。透过技术群的交流，能获取不少工作以外的信息，别人的提问也能触发自己的思考，而大牛之间的讨论往往三言两语就能让自己学到不少知识点。

十二月的时候，先到深圳去参加了猿生态的城市巡回技术沙龙，会后开源中国有提供面试机会，自己却没有足够勇气去参加，其实应该去试一试的，倒不是为了工作，单纯看看自己还欠缺什么也好。12月17日，中国第三届CSS大会在广州举办，难得业界大会选址广州，自然要去参加。嘉宾的分享着实让我这样的新人开了不少眼界。

加入社群最大的收获其实还是见识了更厉害的人，认识了新的朋友。尤其是那些跟我同龄的，又或者跟我同期开始做前端的，甚至还有不少在校大学生，相比起已经在业界具有一定知名度的大牛，和这些更厉害的人交流往往更能让浮躁的自己时刻保持谦虚的态度。我很喜欢《全职高手》里的一句话，并且时常拿来勉励自己。

“在这个赛场上，努力是最不值得拿出来夸口的东西，因为这只是基本，是人人都会做到的，是最底层最渺小的东西。搞清楚这一点，再向高处攀登吧！”

虽然说的是电竞，但其实技术圈又何尝不是如此？一时的领先并不能说明什么，在这个人人都在努力学习，不断提升自己的行业，随时都有被后来者弯道超车的可能。而和那些人比起来，自己的成长速度确实不值一提。

到了2017年，在前端方面我能从公司和团队上学到的东西已经不多了，而当时整个公司，经过几轮人事变动，最终加上我也只有三个前端，并且各自有各自在做的项目，难得有前端技术的交流探讨。不过因为负责了起新项目的整个前端工作，也有了机会实践自己在前端架构上的不少想法，同时也开始正式写PHP做后端CRUD的开发。这之后一直到今天，也算是在稳步成长吧，没有感觉特别明显的进步，但因为保持学习和实践，技术也在一直提升。

思考

其实对任何人来说，入行的第一年总是能收获成长和提升的，区别只是快与慢的差异。当然工作的环境，技术氛围，团队，技术栈等因素也都有影响。我入职的只是一家小公司，在孵化自己的产品，但更多的是接外包的项目，并且算不上是重视技术型的互联网公司，对个人技术的发展必然会有负面阻碍。好在我所在的团队总体上还算OK，至少在初期是利于新人的成长的，我也一直如此认为，但可惜也仅此而已，无法在进阶方面提供额外的成长帮助。并且在没有选择面的情况下，入行初期去到外包公司也能接触到大量的业务场景，只是一定要打听团队情况。

一方面，繁忙的工作，一个接一个的项目，确实很能让一个刚入行的新人大量接触各种类型的业务需求，并且在工作中快速实践运用掌握的基本知识，同时阅读项目其他同事写下的相关代码也是快速上手学习的方法。这个阶段，最关键的还是多看代码，多写业务，以胜任工作为主。

另一方面，在能完成基本的开发工作之后，需要思考的就是如何优化代码，提升编码能力，如何更高效地完成开发工作。而这一点恰恰是需要时间尝试和整理，需要经验来积累和沉淀的东西。外包公司的项目可以帮助积累不同业务场景的经验，但编码能力和架构能力的提升却需要在项目之余好好总结沉淀。此时无尽的外包项目就会是一种阻碍了。

我认为在工作一年之后的今天，对自己而言这是一个非常关键的时间点。是继续日复一日重复做项目来提升技术和业务熟练度，还是跳出这个状态花点时间回过头好好巩固基础，并学习业界的新技术？

选择

这样的状态继续下去，我想我只会局限在目前公司的技术体系下，写着越来越熟练的代码，做着一个又一个项目，但却始终无法停下来好好做一下总结，解决一直以来遗留的问题，尝试新的技术，实践新的想法。也许有的人会说工作之余自学不也一样可以吗？是的，过去我也是如此做的，只是学的东西无法用于工作难免气馁，而且现在这家公司加班还挺严重的，并且有很多不合理安排而导致无意义的加班。加班在所难免，甚至我也常常主动加班，只是现在我越发觉得这是非常影响自我提升效率的事情了。

如今自己早已经完成了第一阶段定下的转型入行的目标，下面我是希望能够在3年之内成为一名优秀的高级前端工程师，但距离这个目标仍然还有很长的路要走。看看自己，页面写不麻溜，JS基础可能比一般人好点，但也不及心目中的高标准，后端技能更是一个渣，此外还有计算机基础，网络基础，数据结构，算法，数学等等这些如地基一般的专业知识需要弥补。倘若自己继续在目前的环境下，不仅无法更加高效地提升前端方面的技术，而且身边的人和事往往还会形成拖累。虽然自己会越来越独当一面，但舞台就那么大，久而久之，自己一定会早早遇到瓶颈而无法突破，最后始终在原地打转。这绝非我希望看到的局面。

穷则思变，想到这里，其实该如何抉择心里早就已经想好了。
